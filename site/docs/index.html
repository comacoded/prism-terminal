PRISM Documentation

PRISM is an agent-native terminal for macOS. This page covers everything it can do, because a lot of it is deliberately quiet until you need it.

Install

  • Download PRISM.dmg from Releases, open it, and drag PRISM to Applications.
  • PRISM is currently ad-hoc signed. On first launch, if macOS blocks it, right-click the app and choose Open once; after that it opens normally.
  • Requirements: macOS 13+. The DMG is a universal binary (Apple silicon and Intel). Linux is on the roadmap.

On first run PRISM starts your login shell (zsh by default), injects its shell integration automatically (details), and restores any previous session.

Updates

PRISM updates itself. It quietly checks for a new version shortly after launch and every few hours while running, and notifies you when one exists (the settings gear also shows a green dot); install it from Settings → Updates (⌘,), which downloads the signed build, verifies it, and relaunches with your session intact. The same section shows your current version and a manual "Check for updates" button.

Interface tour

  • Tab strip (top): tabs with live status dots, group chips, and the + button. The two buttons on the right toggle the artifacts rail and settings.
  • Status dots: violet means an agent is actively working in that tab, blue means an agent is running but idle (often waiting on you), gray is a plain shell, red means the shell exited.
  • The glow: while an agent in the active tab is generating, the window wears an animated gradient border. It stops when the agent goes quiet.
  • Footer: working directory, git branch, last command's exit status and duration, foreground process, and session uptime.
  • Artifacts rail (right): appears when an agent writes files. See Agent supervision.

Tabs & groups

  • ⌘T opens a tab in the current directory. ⌘W closes the focused pane (or the tab if it's the last pane).
  • Switch with ⇧⌘{ / ⇧⌘} or jump directly with ⌘1 through ⌘9.
  • Rename a tab by double-clicking it (or right-click, Rename). An empty name reverts to the shell-reported title. Names persist.
  • Drag tabs to reorder them. A drop indicator shows the target slot.

Groups (Chrome-style)

  • Right-click a tab, then Add to new group, or drop a tab onto an existing group's chip, or drop it between two tabs of a group.
  • Creating a group opens the editor: name it, pick one of nine colors, or ungroup. Right-click a chip to reopen the editor.
  • Click a chip to collapse the whole group into it; click again (or activate a member any other way) to expand. If a group holds every tab, collapsing opens a fresh tab first.
  • Member tabs wear the group color, stay contiguous, and groups persist across launches.

Split panes

  • ⌘D splits right, ⇧⌘D splits down; up to four panes per tab, one direction per tab in v1.
  • Every pane is a full independent shell with its own agent detection, prompt marks, and artifacts.
  • Click a pane to focus it; the focused pane wears an accent border and drives the footer, artifacts rail, search, and prompt navigation. ⌘W closes the focused pane.
  • Resize by dragging the divider between panes.
  • Zoom the focused pane to fill the tab with ⇧⌘↵; toggle again to restore the layout.
  • Broadcast input with ⇧⌘B: whatever you type in the focused pane is sent to every pane in the tab (footer shows "broadcast" while it's on).
  • Close just the split with ⇧⌘W (never closes the whole tab). Right-click a split pane to move it out into its own tab or close it.
  • The tab's dot and the window glow aggregate across panes: if any pane's agent is working, the tab is working.

Palette & search

  • ⌘P opens the command palette: every action plus a "Go to" entry for each open tab. Type to filter, arrows to move, Enter to run.
  • ⌘F searches scrollback with live highlight-as-you-type, a match counter, and scrollbar markers. Enter / Shift+Enter step through matches.
  • ⌘↑ / ⌘↓ jump between shell prompts in scrollback (powered by shell integration).
  • The palette also lists your recent commands (recorded by shell integration, persisted across launches): pick one to run it again in the focused pane.
  • ⌘-click any file path printed in the terminal to open it in your editor (VS Code, Cursor, Zed, or the system default; pick in settings). Paths like src/main.js:42 jump to the line.
  • The palette also holds less-common actions: copy last command output, reveal the current folder in Finder, clear terminal.

Session restore

On relaunch PRISM reopens your workspace: tabs, split layouts, groups, custom names, and each pane's working directory, with fresh prompts ready to go, exactly like iTerm or Terminal reopening your windows.

It does not replay old terminal output. A saved snapshot is a dead picture rather than a live session, and full-screen agents (Claude Code, vim) clear the screen on launch regardless, so replaying it added confusion, not value. The session layout is stored in ~/Library/Application Support/io.nickcoma.prism.spike/session.json and never leaves your Mac.

Themes & settings

Open settings with ⌘, or the gear button.

  • Themes: ten palettes chosen from live preview cards: PRISM, Dark, Light, Dracula, Cyber Wave, Nord, One Dark, Solarized Dark, Gruvbox Dark, Catppuccin Mocha. All render over the native glass. Picking Light restyles the entire chrome, not just terminal colors.
  • Text size: slider from 10 to 20, or ⌘+ / ⌘− anywhere, ⌘0 to reset.
  • Glass tint: how dark (or light, in Light mode) the translucent background is over your wallpaper.
  • Import themes: bring any iTerm .itermcolors file or Ghostty theme in via the Import button; imported themes appear as cards and can be removed with the × on hover.
  • Editor: which editor ⌘-clicked paths open in.
  • Cursor: bar, block, or underline, with a blink toggle.
  • Work glow: turn the agent glow border off if it's not your thing.
  • Shortcuts: the full keyboard reference lives at the bottom of the settings panel.

Everything persists across launches. Reset to defaults with one button.

Agent supervision

PRISM watches the process tree and recognizes CLI agents (Claude Code, Codex) as the foreground process of any pane. Everything below happens automatically; there is nothing to configure.

States and the glow

  • While an agent is actively generating, its tab dot turns violet and the window wears the animated gradient border. Detection reads the live screen for the agent's status line, so the glow holds steadily for the whole turn.
  • When the agent is running but idle, the dot turns blue: usually it's waiting on you.

Artifacts rail

  • While an agent is active, PRISM watches the session's working directory. Every file the agent creates or edits appears in the rail, newest first, with the working folder shown in the header.
  • The rail opens itself when files start landing in the active tab, follows you between tabs, and stays closed for a session once you dismiss it. Files landing in a background tab pulse the rail button instead.
  • Click any entry to reveal the file in Finder. Hovering a row shows two more actions: Quick Look (eye) and git diff, which opens the file's uncommitted changes in an overlay.

Mission control

⌘E shows every tab as a live card: state and time in state, working directory, branch, group, pane count, and the last files touched. Click a card to jump in.

Notifications

  • When an agent needs you: if an agent stops generating while you're in another tab or app, PRISM notifies you, pulses that tab's dot amber, and badges the Dock icon with the count of waiting agents. Everything clears when you return to the tab.
  • Commands that run longer than 15 seconds notify you when they finish, if you're in another tab or app.
  • Programs can send notifications (OSC 9 / OSC 777) and taskbar progress (OSC 9;4), which renders as a hairline progress bar on the tab.

Shell integration

PRISM injects shell integration automatically — zsh via ZDOTDIR, bash via an init file — sourcing your real config untouched before adding hooks. No dotfile edits, and other terminals are unaffected. Nested shells can load it manually: source "$PRISM_INTEGRATION_DIR/prism.zsh" (or prism.bash).

Environment

Every PRISM shell gets TERM=xterm-256color, COLORTERM=truecolor, TERM_PROGRAM=PRISM, TERM_PROGRAM_VERSION, and PRISM_SOCKET.

fish? Everything works, but prompt marks, command telemetry, and the prism function are zsh + bash only today. The footer falls back to slower cwd polling.

Socket API & CLI

PRISM listens on a Unix socket (path in $PRISM_SOCKET, user-only permissions, local machine only). Protocol: one JSON request per line, one JSON response per line. The prism command wraps it in every shell.

~/dev/prism ❯ prism list
{"tabs":[{"index":0,"title":"claude","active":true,"group":"agents",
  "panes":[{"id":1,"cwd":"/Users/you/dev/prism","focused":true,
            "agent":true,"working":true,"exited":false}]}]}

~/dev/prism ❯ prism split row
{"pane":4}
~/dev/prism ❯ prism send 4 "npm test"
~/dev/prism ❯ prism read 4 20
  ✓ 42 passing (1.2s)
CommandJSONDoes
prism list{"cmd":"list"}Every tab and pane with cwd, focus, and live agent state
prism new-tab [dir]{"cmd":"new-tab","cwd":"…"}Open a tab; returns the new pane id
prism split [row|column]{"cmd":"split","dir":"row"}Split the active tab; returns the new pane id
prism send <pane> <text>{"cmd":"send","id":4,"data":"…\n"}Type into any pane (newline appended by the CLI)
prism read <pane> [lines]{"cmd":"read","id":4,"lines":50}Read the last lines of a pane's screen
prism activate <pane>{"cmd":"activate","id":4}Focus that pane's tab and pane
prism notify <text>{"cmd":"notify","title":"…","body":"…"}Post a macOS notification

Raw access without the CLI: echo '{"cmd":"list"}' | nc -U "$PRISM_SOCKET". Agents running inside PRISM can use this to open panes beside themselves, run commands, and read the results.

Troubleshooting

macOS won't open the app

PRISM is ad-hoc signed for now. Right-click the app in Applications and choose Open once.

The glow never appears

The glow tracks Claude Code and Codex as the foreground process of a pane. Check the Work glow toggle in settings, and note it only shows for the active tab; background tabs show the violet dot instead.

Prompt jumping / duration chips don't work

Those need the shell integration (zsh or bash), which applies to shells started inside PRISM. If you exec'd into fish, they're off (see Shell integration).

The global hotkey does nothing

⌃` is registered system-wide at launch; another app may own it already.

Where is my data?

Everything is local: session snapshots and the socket live in ~/Library/Application Support/io.nickcoma.prism.spike/. PRISM makes no network requests, has no telemetry, and no accounts.
